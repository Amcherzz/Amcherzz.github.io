1. 数据丢失

Source到Channel是Put事务，Channel到Sink是Task事务
memoryChannel宕机以及channel满了，Source不再写入，未写入的数据丢失

可能会重复，sink发出，未收到响应会继续发出，导致重复

2. 参数调优

  • source：
    • 增加Source个，增大读取数据的能力
    • batchSize，一次批量运输到Channel的event条数，增大可以提高性能
  • channel
    • memory性能好，容错性差
    • file Channel时dataDirs配置多个不同盘下的目录可以提高性能

Capacity参数： 决定Channel可容纳最大event条数

transactionCapacity参数：决定每次Source往channel里面写的最大event条数和每次Sink从channel里面读的最大event条数

transactionCapacity：需要大于Source和Sink的batchSize参数

  • sink
    • 适当增加sink
    • 适当调大batchSize

3. 术语

  • Agent ：JVM进程，由Source、Channel、Sink组成
  • Source ：数据采集
  • Channel：数据缓存
  • Sink：将数据发送到目的地
  • Event：数据传输单元，有Header（k=v）和Body(字节数组)组成
